# Contribution Guidelines

Thank you for your interest in contributing to the SOC Analyst Journey repository! Here are some guidelines to help you get started:

## How to Contribute
1. Fork the repository.
2. Create a new branch for your feature or bug fix.
3. Make your changes and commit them with a clear message.
4. Push your changes to your fork.
5. Open a pull request describing your changes.

## Code Standards
- Follow the coding conventions of the project.
- Write clear commit messages.

## Issues
- If your changes address an issue, please mention it in your pull request.

Happy contributing!
